# 기초 지식

## 기초 개념
1) returns -> 각 날짜의 각 기업의 주가 수익률을 담고 있는 데이터 필드

2) -returns -> returns의 반대로 베팅
3) alpha -> 데이터 필드와 오퍼레이터의 조합
4) operator -> 데이터 필드 변병 가능
5) PnL -> 알파 벡테스트하면 투자의 결과에 따라 이익과 손해발생 그것을 백테스트 기간에 걸쳐 누적해놓은 것
6) Sharpe -> 얼마나 수익을 안정적으로 얻는가
7) trunover -> 하루마다 평균적으로 얼마나 포지션이 변하는지
8) fitness -> 알파 성과 평가
   
    - sharpe/turnover * returns ^1/2

9) return -> 알파가 얼만큼의 수익을 낼 수 있는가
10) drawdown -> 낙폭을 측정하는 지표
11) margin -> 트레이드 한 양에 비해 얼마만큼의 PnL을 얻었는지
12) neutralization -> 중립화
13) decay -> 알파의 포지션이 변하는 속도 줄여줌
14) setting
    - delay -> 얼마만큼의 지연된 데이터 사용할지
    - decay -> 과거 포지션 얼만큼 반영
    - neutralization -> 포지션이 특정 산업에 몰려 있을때 방지
15) 데이터 셋
    - 제무제표(fundamental)
    - Pv
16) operator
    - 산술 연산자
    - 논리 연산자
    - 시계역 연산자 -> 특정 종목과 관련하여 과거의 d일의 값과 관련된 연산
      - ts_mean(volume, 10)
    - 횡단면 연산자 -> 특정 시점에 대상 종몽들의 값을 비교 또는 처림
      - rank(x) -> 특정 시점 x의 순서를 구해 0에서 1의값을 배분
    - 벡터 연산자 -> 벡터 타입으로 되어있는 데이터 필드를 대표할 수 있는 값으로 바꿔줌
    - 변환 연산자 -> 행렬내의 값을 특정한 연산을 거쳐 변환할 수 있도록 함
    - 그룹 연산자 -> 그룹 데이터 필드를 대표값을 구하거나 중립화하는 연산
17) Pv데이터
    - 가격(price)
      - hign(고가)
      - low(저가)
      - opne(시가)
      - close(종가)
    - 거래량(volume) -> 그날 몇개의 주식이 거래됐는지
18) VWAP(Volume - Weighted - Average - Price)
    - 거래량을 가중 평균한 가격
    - 하루 거래량 평균 가격
    - VWAP + 리버젼
      - VWAP/close > 1 -> 마지막에 급 하강 -> 롱 포지션
      - VWAP.close < 1 -> 마지막에 급 상승 -> 숏 포지션
19) 기술적 분석
    - 과거 데이터를 보고 미래의 가격 움직임 예측
20) CLV(close Location value)
    - 하루동안의 가격 변동 번위에서 종가가 어디에 위치하는지를 나타낸 지표
    - CLV = ((close-low) - (high-close)) / (high-low)
    - 마지막 순간에 시장 심리를 파악할 수 있게하는 지표
    - CLV + 리버젼
      - CLV -> 1 --> 종가가 고가에 가까움 -> 매도
      - CLV -> -1 --> 종가가 저점에 가까움 -> 매수

21) 모멘텀 효과
    - 최근 오른애는 계속 오르고 내린애는 계속 내림
    - 시간 관점에 따라 다름
      - 단기(1개월 미만) -> 리버젼 효과 나타남 
      - 중기(3~12) -> 전통적인 모멘텀
      - 장기(12개월 이상) -> 효과가 점점 사라짐
    - 자산 1년간의 상승분
      - ts_delta(close,250)/tx_delay(close,250)
22) 조건문
    - if volume > adv20 ? 1:0
    - trade_when(entry_condition, alpha, exit_condition)
23) 펀터멘털 데이터
    - 투자자들이 기업의 본질적 가치평가 하는데 사용하는 정보
    - 기업의 실제 사업 현황 객관적으로 파악
    - 기업의 미래 성장 가능성 예측하는 기초자료
      - 재무상태표 -> 기업의 자산, 부채, 자본현황
      - 손익계산서 -> 매출, 비용, 이익정보
      - 현금흐름표 -> 실제 현금 유출입 현황
    - 데이터 업데이트 주기
      - 분기 이상으로 업데이트
      - pv데이터와 달리 불연속적으로 업데이트됨
24) EBITDA
    - 이자, 세금, 감가상각, 무형자산상각 차감전 영업이익
    - 기업의 전반벅인 재무성과 측정
    - 자본구조나 세율이 다른 기업들 비교할때 유용
25) Enterprise Value(EV)
    - 기업의 총 가치
    - 주가 변동애 따라 자주 업데이트 됨
    - 시가총액과 달리 기업의 부채와 현금 포지션 고려
26) OEY(Operating Earnings Yield)
    - operating_income / cap
    - 기업의 수익성 평가

## 개념 응용
1) 가격 - 펀더멘털 결합
  - PV(시가총액) + 펀더멘털(영업이익)
  - 영업이익(operating-income)
    - 분기/연간 단위로 업데이트
    - 기업 실제 수익창출 능력 반영
    - 비지니스의 효율성 측정
  - 시가총액
    - 시장에서 평가하는 기업 가치
    - 일별로 업데이트
    - 주가 * 발행주식
  - 두개의 데이터 비교 방법
    1) 뺄셈
       - ex) 에널리스트 전망 - 실제 기업실적
    2) 나눗셈
       - scale이 다른 데이터를 비교할때 나눠줌으로써 영업이익이 시가총액에 비해 몇 %인지
  - ##### 실습
    - -rank(operating_income/cap) -> 시가총액 대비 영업이익 높은애들은 숏 반대면 롱

2) EBITDA + Enterprise Value
   - 이 둘을 비교함으로써 기업이 수익 창출 능력 대비 저평가 되었는지 고평가 되어있는지 평가 -> 기업가치 배수
   - 나눗셈이 적합한이유
     - 서로다른 규모의 기업 비교
     - 기업가치 배수 만들 수 있음
  - ##### 실습
    - -ts_zscore(enterprise_value/ebitda, 63) -> 고평가된애들 숏
   
3) 시계열 연산자 + 횡단면 연산자
    - 시계열
      - 과거 자신의 점수와 비교
        - ts_rank(x,d) -> 지난 d일 동안의 각 종목별 x값 순위화
        - ts_zscore(x,d) -> 오늘의 x가 d일 간의 평균으로부터 얼마나 떨어져있는지
        - ts_mean(x,d) -> 과거 d일간 x값의 평균
    - 횡단면
      - 다른 친구들과 점수 비교
        - rank(x)
        - zscore(x)
        - winsorize(x, std=4) -> x의 모든 값들이 표준 편차의 배수로 지정된 상한값과 하한값 사이에 들어오도록 극단값제한

    - ##### 실습
      - ts_rank(operating_income/cap, 250) -> 기업의 수익성 평가
4) universe 사용
   - 유니버스 수 작으면 더 유동성이 높아짐 & 높은 시가총액 가진 주식 -> 덩치크고 거래 활발
